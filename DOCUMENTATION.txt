# Django ToDo Application Documentation

## Overview
This is a Django-based ToDo list application that allows users to manage tasks with different statuses (Pending, In Progress, Completed). The application provides a clean web interface for creating, viewing, updating, and deleting tasks.

## Features

### Task Management
- **Create Tasks**: Add new tasks with a title
- **View Tasks**: View all tasks or individual task details
- **Update Status**: Change task status between:
  - Pending (default)
  - In Progress
  - Completed
- **Delete Tasks**: Remove tasks from the list

### Status Workflow
```
PENDING → IN_PROGRESS → COMPLETED
   ↑                      ↓
   ←────── BACKWARDS ─────┘
```

Users can move tasks forward or backward in the workflow at any time.

## Installation & Setup

- Python 3.8+
- Django 4.2.7+

### Setup Instructions

1. **Navigate to the project directory**:
   ```bash
   cd /home/oem/Desktop/todo_project
   ```

2. **Apply database migrations**:
   ```bash
   python manage.py migrate
   ```

3. **Start the development server**:
   ```bash
   python manage.py runserver
   ```

4. **Access the application**:
   Open your browser and navigate to: `http://127.0.0.1:8000`

## URL Endpoints

| URL Pattern | View Function | Description |
|-------------|---------------|-------------|
| `/` | `task_list` | Display all tasks grouped by status |
| `/in-progress/<task_id>/` | `set_in_progress` | Mark task as in progress |
| `/pending/<task_id>/` | `set_pending` | Mark task as pending |
| `/done/<task_id>/` | `mark_done` | Mark task as completed |
| `/task/<task_id>/` | `view_task` | View task details |
| `/delete/<task_id>/` | `delete_task` | Delete a task |
| `/admin/` | Django Admin | Admin interface |

## Data Model

### Task Model
Located in: `todo/models.py`

| Field | Type | Options | Description |
|-------|------|---------|-------------|
| `id` | AutoField | Primary Key | Unique task identifier |
| `title` | CharField | max_length=200 | Task title/name |
| `status` | CharField | choices, default='PENDING' | Current status |
| `created_at` | DateTimeField | auto_now_add=True | Creation timestamp |
| `updated_at` | DateTimeField | auto_now=True | Last update timestamp |

### Status Choices
```python
STATUS_CHOICES = [
    ('PENDING', 'Pending'),
    ('IN_PROGRESS', 'In Progress'),
    ('COMPLETED', 'Completed'),
]
```

## Views Description

### `task_list(request)`
- **Method**: GET/POST
- **Function**: 
  - GET: Displays all tasks grouped by status
  - POST: Creates a new task from form submission

### `set_in_progress(request, task_id)`
- Changes task status to 'IN_PROGRESS'
- Redirects to task list

### `set_pending(request, task_id)`
- Changes task status to 'PENDING'
- Redirects to task list

### `mark_done(request, task_id)`
- Changes task status to 'COMPLETED'
- Redirects to task list

### `view_task(request, task_id)`
- Displays detailed information about a single task
- Shows title, status, creation date, and last update date
- Provides action buttons for status changes

### `delete_task(request, task_id)`
- Deletes the specified task from the database
- Redirects to task list with confirmation

## Database Schema

### Tasks Table
```sql
CREATE TABLE todo_task (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title VARCHAR(200) NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'PENDING',
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```
